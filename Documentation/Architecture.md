## 核心系统详解

### 1. GameManager（游戏管理器）

**职责**：
- 初始化所有子系统
- 协调各系统间的通信
- 管理游戏生命周期
- 处理离线收益

**关键方法**：
- `Initialize()` - 系统初始化
- `HandleOfflineEarnings()` - 离线收益计算
- `GetPlayerData()` - 获取玩家数据

### 2. LevelManager（关卡管理器）

**职责**：
- 管理1000个关卡数据
- 生成关卡怪物波次
- 计算关卡难度和奖励
- 追踪玩家进度

**特性**：
- 20个章节（每章50关）
- 难度逐步递进
- 奖励指数增长（1.05倍增长率）

**公式**：
```
Gold = 100 × 1.05^(level-1)
Exp = 50 × 1.05^(level-1)
Difficulty = 1.0 + (level-1) × 0.01
```

### 3. BattleManager（战斗管理器）

**职责**：
- 执行自动战斗逻辑
- 计算伤害和防御
- 管理战斗状态
- 处理战斗结果

**战斗流程**：
1. 玩家队伍攻击
2. 敌方受伤
3. 移除死亡敌人
4. 检查胜负条件
5. 重复至战斗结束

### 4. PlayerData（玩家数据）

**管理内容**：
- 玩家等级
- 金币和钻石
- 经验值
- 招募的英雄列表
- 当前关卡

**存储方式**：
- 使用PlayerPrefs存储JSON
- 自动保存机制
- 支持存档导入导出

### 5. 角色系统（Hero/Monster）

**英雄系统**：
- 50+个可招募英雄
- 5个稀有度等级（1-5星）
- 性别差异（重点是美女角色）
- 技能系统
- 装备系统
- 升级机制

**怪物系统**：
- 200+个山海经神兽
- 等级化难度
- 特殊技能
- 稀有掉落

## 数据流向

```
玩家操作
   ↓
 UIManager → GameManager → 相应管理器
                  ↓
            业务逻辑处理
                  ↓
            PlayerData 更新
                  ↓
            本地存储
                  ↓
            UI 刷新
```

## 扩展点

### 1. 新英雄添加
在 `HeroConfig.cs` 中添加新的HeroData实例

### 2. 新关卡
`LevelManager` 自动生成1000个关卡

### 3. 新技能
创建 `SkillConfig.cs` 扩展技能系统

### 4. 新道具
创建 `ItemConfig.cs` 管理可装备道具

### 5. 新UI界面
在 `Assets/Scripts/UI/` 下创建新的UI管理脚本

## 性能优化

### 1. 对象池
- 复用战斗特效
- 复用浮窗文字

### 2. 异步加载
- 3D模型延迟加载
- 场景异步切换

### 3. 内存管理
- 及时释放未使用资源
- 使用Resources.UnloadUnusedAssets()
